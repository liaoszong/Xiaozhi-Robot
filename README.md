# Xiaozhi-Robot

基于小智 ESP32 与 STM32F103RCT6 的语音交互智能小车项目。

## 项目简介

本项目将小智 ESP32 的语音交互能力与 STM32F103RCT6 小车底盘控制系统结合，实现通过自然语言控制小车运动，并在此基础上扩展动作序列和场景任务演示。系统整体采用“ESP32-S3 负责 AI 交互与任务编排，STM32F103RCT6 负责底盘实时控制”的分层结构，适合作为嵌入式、语音交互与小车控制结合的工程实践项目。

当前项目主要完成了语音唤醒、AI 对话、基础运动控制、UART 安全帧通信、动作序列执行、部分 mission 场景任务、OLED 状态显示、循迹与视觉识别模块验证等内容。循迹、视觉识别和返回出发点等功能仍属于演示和扩展验证阶段，尚未形成完整的自主导航闭环。

## 系统架构

系统分为两层：

- **ESP32-S3 小智 AI 端**：负责语音唤醒、语音识别、AI 对话、自然语言理解、MCP 工具调用、任务编排，并通过 UART 向 STM32 发送小车控制帧。
- **STM32F103RCT6 底盘端**：负责 UART 指令解析、电机 PWM 控制、OLED 状态显示、循迹传感器采集、九轴陀螺仪 yaw 辅助控制和底盘动作执行。

两端通过 UART 通信。ESP32 侧把 AI 解析出的动作转换为 `CAR:<命令字符>\n` 安全帧，STM32 侧只在识别到合法帧后执行动作，从而减少普通串口日志或错误字节直接触发小车运动的可能性。

## 主要功能

- 语音控制小车前进、后退、左转、右转和停止；
- 支持 `CAR:X` 安全帧通信，降低启动日志或误码导致的误触发风险；
- 支持 `self.car.control` 单动作控制，适合“前进”“停止”等简单指令；
- 支持 `self.car.sequence` 动作序列，适合“先前进 3 秒，再右转，最后停止”等复合指令；
- 支持 `self.car.mission` 场景任务扩展，例如巡逻、展示、模拟送药、自检、返回等任务演示；
- STM32 侧支持 OLED 状态显示、电机控制、循迹模块测试和九轴陀螺仪 yaw 辅助控制；
- 视觉识别和循迹目前属于扩展验证功能，已完成模块级测试，但尚未形成成熟自主导航系统。

## 通信协议

ESP32 发送给 STM32 的基本格式为：

```text
CAR:<命令字符>\n
```

当前核心控制命令如下：

| 安全帧 | 命令字符 | 动作 |
| --- | --- | --- |
| `CAR:A\n` | `A` | 前进 |
| `CAR:E\n` | `E` | 后退 |
| `CAR:G\n` | `G` | 左转 |
| `CAR:C\n` | `C` | 右转 |
| `CAR:Z\n` | `Z` | 停止 |

代码中还保留了若干扩展动作字符，用于兼容或后续扩展，例如左前、右前、左后、右后等组合动作。实际演示时建议优先使用上表中的核心安全帧，并以 STM32 侧 `Bluetooth.h` 和 `main.c` 中的命令解析逻辑为准。

## 快速开始

### ESP32 侧

ESP32 侧基于 ESP-IDF 工程组织，具体编译和烧录方式请根据本机 ESP-IDF 环境、目标开发板和 `menuconfig` 配置执行。常见流程包括：

```text
idf.py set-target esp32s3
idf.py menuconfig
idf.py build
idf.py flash monitor
```

本仓库不提交本机生成的 `sdkconfig`、`sdkconfig.old` 和 `build/` 目录。需要重新编译时，请根据实际硬件重新配置。

### STM32 侧

STM32 侧可根据项目文件使用 STM32CubeMX、Keil MDK 或 STM32CubeIDE 打开和编译。常见关注点包括：

- 确认目标芯片为 STM32F103RCT6；
- 确认 USART3 与 ESP32 侧 UART 参数一致；
- 确认电机 PWM、方向控制、OLED、循迹传感器和九轴陀螺仪连接与代码配置一致；
- 编译输出目录、Keil 中间文件和本机 IDE 配置不会提交到仓库。

## 当前状态与局限

- 当前系统主要完成语音控制、动作序列、任务演示和实物联调；
- 返回出发点目前是基于动作序列的近似返回，不是基于地图、里程计或 SLAM 的精确定位；
- 循迹和视觉识别已完成模块验证，但尚未与语音任务系统形成成熟的自主导航闭环；
- 电机一致性、底盘结构安装、麦克风安装位置、地面摩擦和 yaw PID 参数仍会影响运动效果；
- mission 场景任务适合用于演示任务编排能力，实际应用仍需要进一步增加定位、避障和状态反馈机制。
